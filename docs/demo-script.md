# Demo Script

1. Open SpriteCraft Studio and show the asset families in the left rail.
2. Select Character, Monster, and Tile to demonstrate category-aware shell state.
3. Adjust style, size, frame count, palette, seed, transparency, and Style Lock controls.
4. Switch between single preview, sprite sheet, checkerboard, and animation modes.
5. Generate one asset with Doubao Seedream and show the newest asset selected automatically.
6. If the API call fails, show the queue error and explain that the app no longer substitutes local mock results.
7. Enable Style Lock, generate a matching monster or prop, and point out the inherited style tags.
8. Pick an asset in the inspector and export PNG, sprite sheet, metadata JSON, or the engine ZIP.
9. Explain how the PR sequence moved from prototype shell to real Doubao Seedream generation.

Judging points to mention:

- Product completeness: editor shell, prompt controls, Doubao Seedream generation, preview modes, inspector, and real export actions.
- Innovation: style lock and engine-oriented export bridge image generation to game production.
- Code quality: typed domain model, isolated generator service, reusable UI components, CI.
- Process quality: six scoped PRs with runnable checkpoints and clear commit intent.
